1. Mỗi lần cập nhật Backend
Trên Windows push code lên git
cd /d/GymSupport/EXE201_GymSupport
git status
Chỉ add file cần thiết, không dùng git add .:
git add GymSup-BE
git commit -m "Update backend"
git push origin main
Trên EC2
ssh -i gymsupport-key.pem ubuntu@46.51.222.230
cd /opt/gymsupport/source
git pull origin main
Build + chạy lại container:
cd /opt/gymsupport/source/GymSup-BE
sudo docker compose up -d --build
Xem log:
sudo docker logs gymsup-be-api-1 --tail 50
Test:
curl -i https://api.gsfitness.id.vn/api/subscriptions/plans
Nếu code mới cần thêm biến môi trường:
sudo nano /etc/gymsupport.env
sudo docker compose restart
(secrets đọc từ /etc/gymsupport.env, không cần rebuild image khi chỉ đổi biến môi trường)

2. Mỗi lần cập nhật Admin React
Trên Windows:
cd D:\GymSupport\EXE201_GymSupport\gym_admin_dashboard
npm run build
Upload dist:
scp -i gymsupport-key.pem -r .\dist\* ubuntu@46.51.222.230:/tmp/admin-dist/
Trên EC2:
sudo rm -rf /var/www/admin-dashboard/*
sudo cp -r /tmp/admin-dist/* /var/www/admin-dashboard/
sudo chown -R www-data:www-data /var/www/admin-dashboard
rm -rf /tmp/admin-dist
Không cần restart backend hoặc Nginx. Mở lại:
https://admin.gsfitness.id.vn
Nếu trình duyệt còn bản cũ, nhấn Ctrl+F5.

3. Mỗi lần cập nhật Flutter
Flutter không upload lên EC2. Build app:
cd D:\GymSupport\EXE201_GymSupport\gym_support
flutter clean
flutter pub get
flutter build apk --release `
  --dart-define=BACKEND_URL=https://api.gsfitness.id.vn
Đưa lên CH Play:
Nhớ tăng version trong pubspec.yaml (vd 1.0.0+4 -> 1.0.0+5) trước khi build.
flutter build appbundle --release `
  --dart-define=BACKEND_URL=https://api.gsfitness.id.vn
File AAB:
build\app\outputs\bundle\release\app-release.aab
Nếu thay đổi cả backend và Flutter, hãy deploy backend trước, test API thành công rồi mới build Flutter.
